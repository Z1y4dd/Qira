# Plan 04: Child Profile CRUD

## Goal

Real Service Layer bodies for `listChildProfiles`, `createChildProfile`, `updateChildProfile` (new function added), and `deleteChildProfile`. Authenticated UI under `/profiles/new` (create) and `/profiles/[childId]/edit` (edit). Both forms in Arabic, NFC-normalized at the Server Action boundary, age range 5–12 enforced by Zod + DB check. Delete moves to Slice 6 where it lives alongside the manage page + type-the-name modal.

## Demo gate

`pnpm dev`, sign in, `/profiles/new` — form renders with: display name (Arabic), age (5–12), grade band (4-option radio). Submit → row appears in `child_profiles`. Visit `/profiles/[childId]/edit` — form pre-fills, submit → row updates. RLS verified: another parent (manually via curl + their JWT) cannot SELECT this row. `pnpm test:run` reports `tests/unit/profiles-service.test.ts` green (the Zod-only unit tests; integration runs in Slice 6 cross-user E2E).

## Requirements covered

- **PROF-01** — Create child profile (display name, age 5–12, grade band).
- **PROF-02** — Edit child profile.
- **PROF-03** — Delete schema-level cascade is already in place (Phase 1); the UI for it ships in Slice 6.
- **PROF-06** — All queries filter by `parent_id` (Service Layer); RLS enforces at DB.

- **Task 4.2 — `listChildProfiles` Service body**
  - **Files:** `src/services/profiles.ts`
  - **Action:**
    - Replace stub. Signature: `export async function listChildProfiles(supabase: SupabaseClient): Promise<ChildProfile[]>`.
    - Body:
      1. `const parent = await requireParent(supabase);`
      2. `const rows = await db.select().from(childProfiles).where(eq(childProfiles.parentId, parent.parentId)).orderBy(childProfiles.createdAt);`
      3. Map rows to `ChildProfile[]` (cast `gradeBand`, set `assignedLevel: null` for now — Phase 3 will populate it from `currentLevelId`).
    - **Note:** Phase 1 schema has `child_profiles.current_level_id` — Phase 3 will populate it. For now `assignedLevel` is always null in the returned object.
  - **Verification:** Cross-user check: sign in as Parent A, list returns A's children only.
  - **Commit:** `feat(services/profiles): real listChildProfiles body filtered by parent_id`

- **Task 4.3 — `updateChildProfile` Service body (NEW function)**
  - **Files:** `src/services/profiles.ts`
  - **Action:**
    - Add to the exports of `profiles.ts`: `export async function updateChildProfile(supabase: SupabaseClient, childId: ChildId, input: UpdateChildProfileInput): Promise<ChildProfile>`.
    - Body:
      1. `const parent = await requireParent(supabase);`
      2. Parse `input` with `UpdateChildProfileInput` Zod schema.
      3. `const normalized = nfc(parsed.data, ['displayName']);`
      4. `const [row] = await db.update(childProfiles).set({ displayName: normalized.displayName, age: normalized.age, gradeBand: normalized.gradeBand }).where(and(eq(childProfiles.id, childId), eq(childProfiles.parentId, parent.parentId))).returning();`
      5. If `row` is undefined (no match — either wrong child or another parent's child), throw `new AuthError('UNAUTHENTICATED')` (the RLS already blocks the update, but a defensive throw makes the failure mode explicit).
      6. Return mapped `ChildProfile`.
    - **Important:** the `eq(childProfiles.parentId, parent.parentId)` belt-and-suspenders matches the Service Layer's defense-in-depth contract from Phase 1 architecture (RLS is one layer, Service is another).
  - **Verification:** `pnpm tsc --noEmit` clean.
  - **Commit:** `feat(services/profiles): updateChildProfile with defense-in-depth parent_id filter`
