# Plan 02: Sign-Up + Sign-In + Verify-Email

## Goal

Ship the three public auth routes (`/sign-up`, `/sign-in`, `/verify-email`) plus the OAuth callback (`/auth/callback`), all rendered in Arabic, all RTL, all going through `<ArabicText>` for every literal. Email/password is the primary path; the Google OAuth button renders only when `NEXT_PUBLIC_GOOGLE_OAUTH_ENABLED === 'true'`. After successful sign-up the parent lands at `/verify-email` with a "check your inbox" message + "resend email" button. The hard-verification gate from Slice 1 then enforces unverified → `/verify-email` on any authenticated-route attempt.

## Demo gate

`pnpm dev` and `/sign-up` — form renders in Arabic, accepts email + password, on submit the parent is redirected to `/verify-email`. Inbox shows a Supabase verification email (default sender) with a link → click → Supabase exchanges code → redirect to `/auth/callback` → redirect to `/choose-child` (which 404s for now; Slice 5 ships it). `/sign-in` — same form shape, on submit redirects to `/choose-child`. With `NEXT_PUBLIC_GOOGLE_OAUTH_ENABLED=true` in `.env.local`, the Google button renders on both pages.

## Requirements covered

- **AUTH-01** — Email/password sign-up via Supabase Auth.
- **AUTH-02** — Google OAuth (scaffolded; user enables the provider in Supabase dashboard + flips the env flag).
- **AUTH-03** — Email verification required (the email is sent by Supabase on signup; the gate from Slice 1 enforces it).
- **AUTH-05** — Session is set in cookies by `signInWithPassword` / `exchangeCodeForSession`; persists across refresh.

## Success criteria touched

- **#1** — sign-up + sign-in + Google OAuth (mocked callback) + session-persists-on-refresh all live; password reset is Slice 3.

- **Task 2.2 — `/sign-up` page + Server Action + Client form**
  - **Files:** `src/app/(auth)/sign-up/page.tsx`, `src/app/(auth)/sign-up/actions.ts`, `src/app/(auth)/sign-up/sign-up-form.tsx`
  - **Action:**
    - `page.tsx` (Server Component): renders `<ArabicText as="h1" size="reader">إنشاء حساب</ArabicText>` + the Client form + a link to `/sign-in`.
    - `actions.ts`: `'use server'`, exports `signUpAction(_prev, formData)` (RHF-style 3-arg action). Parses with `SignUpInput`, creates `supabase = createClient(await cookies())`, calls `supabase.auth.signUp({ email, password, options: { emailRedirectTo: \`${process.env.NEXT_PUBLIC_SITE_URL}/auth/callback\` } })`. On Supabase error returns `{ error: { _form: [errorMessageAr(error)] } }` (translate common Supabase errors to Arabic). On success: `redirect('/verify-email')`.
    - `sign-up-form.tsx` (Client Component, `'use client'`): RHF + `zodResolver(SignUpInput)` + shadcn `Form`. Email + password fields with Arabic labels. Submit button "إنشاء الحساب". `<GoogleButton>` below `<Separator>` if `NEXT_PUBLIC_GOOGLE_OAUTH_ENABLED === 'true'`. Errors render in Arabic via `<ArabicText size="caption" className="text-destructive">`.
    - Helper `errorMessageAr(supabaseError)` maps common Supabase error codes to Arabic: `email_address_invalid` → "بريد إلكتروني غير صالح"; `user_already_exists` → "هذا البريد مسجّل بالفعل — جرّب تسجيل الدخول"; generic → "حدث خطأ، حاول مرة أخرى".
  - **Verification:** Manual `pnpm dev`: visit `/sign-up`, validation fires in Arabic, submit posts the action, Supabase email arrives, redirect to `/verify-email` confirmed.
  - **Commit:** `feat(auth/sign-up): /sign-up page + Server Action + Arabic RHF form`

- **Task 2.4 — `/verify-email` page + resend Server Action**
  - **Files:** `src/app/(auth)/verify-email/page.tsx`, `src/app/(auth)/verify-email/actions.ts`
  - **Action:**
    - `page.tsx` (Server Component): reads `cookies()`, `createClient`, `supabase.auth.getUser()`. If user is null → redirect to `/sign-in`. If `email_confirmed_at` is non-null → redirect to `/choose-child`. Otherwise render: `<ArabicText size="reader">تحقّق من بريدك الإلكتروني</ArabicText>` + "أرسلنا لك رسالة تأكيد إلى \<email\>." + a "أعد إرسال الرسالة" button bound to the resend Server Action + a "تسجيل الخروج" small link to `/sign-out` (Server Action that does `supabase.auth.signOut()` + redirect to `/`).
    - `actions.ts`: `resendVerificationAction` → `supabase.auth.resend({ type: 'signup', email: user.email! })`. Returns `{ ok: true }` or `{ error: 'حدث خطأ، حاول مرة أخرى' }`. Client form shows a success toast / inline message on `ok: true`.
  - **Verification:** Manual: after sign-up, the page renders with the parent's email; clicking resend triggers another Supabase email.
  - **Commit:** `feat(auth/verify-email): /verify-email landing page + resend Server Action`

- **Task 2.5 — `/auth/callback` route handler (OAuth + email-link exchange)**
  - **Files:** `src/app/auth/callback/route.ts`
  - **Action:**
    - Route Handler (`GET`). Reads `?code=...` from `request.url`. Creates `supabase = createClient(await cookies())`, calls `await supabase.auth.exchangeCodeForSession(code)`. On success → `NextResponse.redirect(new URL('/choose-child', request.url))`. On error → redirect to `/sign-in?error=callback_failed`.
    - **Critical:** the `exchangeCodeForSession` call uses `getUser()` semantics internally (the server validates with Supabase Auth) — does NOT trigger the `getSession` ban.
  - **Verification:** Click verification email link → redirects to `/auth/callback?code=...` → redirects to `/choose-child` (404 OK for now). `tests/invariants/auth-getsession-ban.test.ts` still green.
  - **Commit:** `feat(auth/callback): OAuth + email-link code exchange route handler`

- **Task 2.6 — Google OAuth button (feature-flagged)**
  - **Files:** `src/components/auth/google-button.tsx`, `.env.example`
  - **Action:**
    - Client Component. Renders `null` if `process.env.NEXT_PUBLIC_GOOGLE_OAUTH_ENABLED !== 'true'`.
    - Otherwise renders a `<Button variant="outline" size="lg">` with a Google `<svg>` icon (inline SVG, no extra dep) + `<ArabicText size="ui">المتابعة باستخدام Google</ArabicText>`.
    - On click: `createClient().auth.signInWithOAuth({ provider: 'google', options: { redirectTo: \`${window.location.origin}/auth/callback\` } })`.
    - Add to `.env.example`: `# Set to 'true' to enable Google OAuth button on /sign-in and /sign-up\n# NEXT_PUBLIC_GOOGLE_OAUTH_ENABLED=true`.
  - **Verification:** Without the env flag, no Google button renders. With the flag (manually set in `.env.local`), the button renders and tapping it redirects to Google's OAuth consent screen (assuming the user has configured the Google Cloud project — if not, Supabase returns a `provider_not_enabled` error which is fine for the smoke test).
  - **Commit:** `feat(auth): Google OAuth button gated by NEXT_PUBLIC_GOOGLE_OAUTH_ENABLED`
