# Plan 06: Privacy + Export + Delete + Cross-User E2E

## Goal

Close the parent-data-rights surfaces (Arabic privacy notice + per-child JSON export + type-the-name hard delete with cascade) and the Phase 2 verification gate (Playwright cross-user E2E that creates two parents via admin API and asserts no session bleed). Provide the four Arabic Supabase email-template drafts as a separate doc the user pastes into the Supabase dashboard.

## Demo gate

`pnpm dev`, sign in, pick a child, visit `/profiles/[childId]/manage` — page shows: profile summary + "تصدير بيانات الطفل" button (downloads JSON) + "حذف الملف الشخصي" button. Clicking delete opens a modal: "اكتب اسم الطفل ([name]) للتأكيد" with a text field. Typing the exact display name enables the destructive button; clicking it triggers the Server Action, the row is gone, `attempts` + `attempt_answers` for this child are gone (cascade), redirected to `/choose-child`. `/privacy` renders the Fusha placeholder text + link from sign-up/sign-in pages. Playwright cross-user E2E green: Parent A signs up, creates child Ahmad; Parent B signs up in fresh context, sees zero children; manually forging `qira_active_child=<Ahmad's UUID>` cookie still shows zero children (RLS blocks).

**autonomous: false** — Tasks 6.6 and 6.7 require human action: pasting the Arabic email templates into the Supabase dashboard, and configuring `SUPABASE_SERVICE_ROLE_KEY` as a GitHub Secret for CI E2E.

## Tasks

- **Task 6.1 — `src/services/data-export.ts` Service**
  - **Files:** `src/services/data-export.ts`
  - **Action:**
    - New Service module. Export `exportChildData(supabase: SupabaseClient, childId: ChildId): Promise<{ filename: string; json: string }>`.
    - Body:
      1. `const parent = await requireParent(supabase);`
      2. Fetch child profile (filter by `id === childId && parentId === parent.parentId`). If not found → throw `AuthError('UNAUTHENTICATED')`.
      3. Fetch all `attempts` for this child + all `attempt_answers` for those attempts.
      4. Build the export object:
         ```ts
         const data = {
           exportedAt: new Date().toISOString(),
           parent: { email: parent.email },
           child: { id: child.id, displayName: child.displayName, age: child.age, gradeBand: child.gradeBand },
           attempts: attempts.map(a => ({ id: a.id, kind: a.kind, score: a.score, startedAt: a.startedAt, finishedAt: a.finishedAt })),
           attemptAnswers: answers.map(a => ({ id: a.id, attemptId: a.attemptId, questionId: a.questionId, chosenChoiceId: a.chosenChoiceId, isCorrect: a.isCorrect, answeredAt: a.answeredAt })),
         };
         ```
      5. Return `{ filename: \`qira-export-${child.displayName.replace(/\\s+/g, '-')}-${new Date().toISOString().slice(0,10)}.json\`, json: JSON.stringify(data, null, 2) }`.
    - File header documents the contract: "no markup, no HTML, pure data dump."
  - **Verification:** `pnpm tsc --noEmit` clean. Purity test stays green.
  - **Commit:** `feat(services/data-export): exportChildData service with parent + child + attempts + answers`

- **Task 6.2 — `deleteChildProfile` Service body (real)**
  - **Files:** `src/services/profiles.ts`
  - **Action:**
    - Replace stub. Signature: `export async function deleteChildProfile(supabase: SupabaseClient, args: { childId: ChildId; confirmName: string }): Promise<void>`.
    - Body:
      1. `const parent = await requireParent(supabase);`
      2. Fetch the child to read its display name. If not found → throw `AuthError('UNAUTHENTICATED')`.
      3. Belt-and-suspenders: if `args.confirmName !== child.displayName` → throw `new Error('DELETE_NAME_MISMATCH')`. (The UI is the first gate; this is the second.)
      4. `await db.delete(childProfiles).where(and(eq(childProfiles.id, args.childId), eq(childProfiles.parentId, parent.parentId)));`
      5. Schema `onDelete: 'cascade'` (Phase 1) handles `attempts` + `attempt_answers`.
  - **Verification:** Cross-checked in Slice 6.6 by the E2E that asserts `attempts` is gone after delete.
  - **Commit:** `feat(services/profiles): real deleteChildProfile with name-confirmation + cascade-relying delete`
