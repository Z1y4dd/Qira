# Plan 03: Password Reset

## Goal

Two-page password reset flow: `/reset-password/request` accepts an email and sends the Supabase recovery email. The email link lands at `/reset-password?code=...`. That page exchanges the code for a short-lived session, then shows a new-password form. Submit calls `supabase.auth.updateUser({ password })` and redirects to `/sign-in?reset=ok` with a one-shot success banner.

## Demo gate

`pnpm dev`, `/reset-password/request` — enter the email of a signed-up parent, submit. Inbox receives Supabase recovery email. Click link → lands at `/reset-password?code=...` → renders "اختر كلمة مرور جديدة" form. Submit → redirected to `/sign-in?reset=ok` which shows a green "تم تحديث كلمة المرور — سجّل الدخول الآن" banner. Sign in with the new password works.

## Tasks

- **Task 3.1 — `/reset-password/request` page + Server Action**
  - **Files:** `src/app/(auth)/reset-password/request/page.tsx`, `src/app/(auth)/reset-password/request/actions.ts`
  - **Action:**
    - Page (Server Component): renders `<ArabicText as="h1" size="reader">نسيت كلمة المرور؟</ArabicText>` + helper text "أدخل بريدك الإلكتروني وسنرسل لك رابطاً لإعادة التعيين." + a Client form with email field + submit button "أرسل الرابط" + link back to `/sign-in`.
    - Action: parses with `ResetRequestInput` Zod schema. Calls `supabase.auth.resetPasswordForEmail(email, { redirectTo: \`${process.env.NEXT_PUBLIC_SITE_URL}/reset-password\` })`. Returns `{ ok: true }` on success or `{ error: 'بريد غير معروف' }` on error.
    - **UX:** On success, the page replaces the form with `<ArabicText size="reader">تحقّق من بريدك الإلكتروني — أرسلنا لك رابط إعادة التعيين</ArabicText>` (do not auto-redirect — let the parent close the tab).
  - **Verification:** Manual flow against your Supabase project. Supabase rate-limits recovery emails to 60s per email — back-to-back submits will silently no-op the second one (Supabase returns success either way to prevent enumeration; that's the intended behavior).
  - **Commit:** `feat(auth/reset-password): request page + resetPasswordForEmail Server Action`

- **Task 3.2 — `/reset-password` page (code exchange + form)**
  - **Files:** `src/app/(auth)/reset-password/page.tsx`, `src/app/(auth)/reset-password/actions.ts`, `src/app/(auth)/reset-password/reset-form.tsx`
  - **Action:**
    - `page.tsx` (Server Component, async):
      1. Read `?code=...` from `searchParams`. If missing → render an error state `<ArabicText size="reader">الرابط غير صالح أو منتهي الصلاحية</ArabicText>` + a "اطلب رابطاً جديداً" button linking back to `/reset-password/request`.
      2. If present: `const supabase = createClient(await cookies()); const { error } = await supabase.auth.exchangeCodeForSession(code);`. If error → same error state as above.
      3. If success: render the new-password Client form.
    - `actions.ts`: `updatePasswordAction(_prev, formData)` parses `ResetApplyInput` (just `{ password }`), calls `supabase.auth.updateUser({ password })`. On success → `redirect('/sign-in?reset=ok')`. On error: `same_password` → "يجب أن تختار كلمة مرور مختلفة"; weak password → reuse the same Zod min-length message.
    - `reset-form.tsx` (Client): RHF + Zod, one password field + confirm field. Submit "حفظ كلمة المرور الجديدة".
    - Update `/sign-in/page.tsx` (from Slice 2) to read `?reset=ok` and render a green banner above the form: `<ArabicText size="ui" className="text-green-700">تم تحديث كلمة المرور — سجّل الدخول الآن</ArabicText>`.
  - **Verification:** Full flow against the real Supabase project. New password works on sign-in.
  - **Commit:** `feat(auth/reset-password): code-exchange page + new-password form + sign-in success banner`
