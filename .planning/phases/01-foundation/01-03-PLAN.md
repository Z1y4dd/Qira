# Plan 03: Drizzle Schema + RLS

## Goal

Define the eight v1 entities (`parents`, `child_profiles`, `levels`, `texts`, `questions`, `choices`, `attempts`, `attempt_answers`) in a single Drizzle schema file with `crudPolicy()` co-located on every table; generate and apply the first migration against the local Supabase emulator and (in Slice 6) against the cloud project; seed 20 `levels` rows; and prove RLS coverage with a Vitest test that walks every exported `pgTable` via `getTableConfig`.

## Demo gate

`pnpm supabase start` brings up the local emulator. `pnpm drizzle-kit generate` produces a migration in `drizzle/migrations/0000_initial.sql` whose body includes `CREATE TABLE`, `ALTER TABLE ... ENABLE ROW LEVEL SECURITY`, and `CREATE POLICY` statements for all 8 tables. `pnpm drizzle-kit migrate` applies it. `pnpm db:seed` inserts 20 rows into `levels` and exits cleanly. `pnpm test:run` includes the new `tests/invariants/rls-coverage.test.ts` — every table reports `policies.length > 0`. A psql query against the local DB (`SELECT tablename FROM pg_tables WHERE schemaname = 'public' AND tablename NOT IN (SELECT tablename FROM pg_policies WHERE schemaname = 'public');`) returns zero rows.

## Requirements covered

- **FOUND-05** — Drizzle schema with all 8 entities applied to Supabase Postgres.
- **FOUND-06** — RLS via `crudPolicy()` on every table + CI test that fails the build if any table is missing RLS.
