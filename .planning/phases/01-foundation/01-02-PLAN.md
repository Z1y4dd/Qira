# Plan 02: Lint + Invariant CI

## Goal

Stand up the eight static-invariant gates as CI checks. Biome handles general JS/TS lint+format; bash scripts handle the RTL and force-dynamic gaps Biome doesn't cover; Vitest hosts the source-scan tests and is the future home for the RLS-coverage and service-layer-purity invariants (added in Slices 3 and 4). The GitHub Actions workflow runs everything on every push and pull request. Slice exit proof: a deliberately-bad branch (introducing `ml-2`) fails CI; `main` passes.

## Demo gate

A short demo branch — `demo/bad-rtl` — adds `ml-2` to `app/[locale]/page.tsx`, opens a PR, and the GitHub Actions run **fails** on the `lint-rtl` step. The same workflow run on `main` (with `ml-2` reverted) is fully green: Biome ci, tsc, lint-rtl, lint-force-dynamic, lint-arabic-wrapper (warn-only), vitest run, all in under 90 seconds.

- **Task 2.2 — Add `scripts/lint-rtl.sh` ripgrep gate for physical-direction utilities**
  - **Files:** `scripts/lint-rtl.sh`
  - **Action:**
    - Create the script exactly per RESEARCH §B Option (a). Make it executable: `chmod +x scripts/lint-rtl.sh`.
    - Recommended exact pattern (ripgrep):
      ```bash
      #!/usr/bin/env bash
      set -euo pipefail
      PATTERN='\b(ml|mr|pl|pr|border-l|border-r|rounded-l|rounded-r|rounded-tl|rounded-tr|rounded-bl|rounded-br|left|right|float-left|float-right|clear-left|clear-right|text-left|text-right)-[0-9a-z\[\]]+\b|(\bml-|\bmr-|\bpl-|\bpr-|\btext-left\b|\btext-right\b|\bfloat-left\b|\bfloat-right\b)'
      if rg --type-add 'web:*.{tsx,ts,jsx,js,css}' -t web --no-messages "$PATTERN" src/ app/ 2>/dev/null; then
        echo "ERROR: Physical-direction Tailwind utility found. Use logical properties (ms-*, pe-*, text-start, etc.)"
        exit 1
      fi
      echo "OK: No physical-direction utilities."
      ```
    - **Note on direction-neutral utilities** (RESEARCH §B): `mx-*`, `my-*`, `px-*`, `py-*`, `border-x-*`, `border-y-*`, `inset-x-*`, `inset-y-*` are *permitted* — the regex above does NOT match them (they share the `m`/`p`/`border`/`inset` prefix but with `x`/`y` not `l`/`r`).
    - Test the regex locally: introduce `ml-2` in `app/[locale]/page.tsx`, run `bash scripts/lint-rtl.sh`, observe exit code 1 and an error line pointing to the file. Revert.
  - **Verification:** `bash scripts/lint-rtl.sh` on the clean tree exits 0 with `OK: No physical-direction utilities.`. After introducing `ml-2` somewhere, it exits 1.
  - **Commit:** `feat(ci): add scripts/lint-rtl.sh ripgrep gate for physical-direction Tailwind utilities`

      ```ts
      import { glob } from 'tinyglobby';
      import { readFile } from 'node:fs/promises';
      import { expect, test } from 'vitest';

      const FORBIDDEN = /\b(ml|mr|pl|pr|text-left|text-right|float-left|float-right|border-l|border-r|rounded-(l|r|tl|tr|bl|br))-?\w*/g;

      test('no physical-direction Tailwind utilities in source', async () => {
        const files = await glob(['src/**/*.{ts,tsx,css}', 'app/**/*.{ts,tsx,css}']);
        const offenders: Array<{ file: string; matches: string[] }> = [];
        for (const f of files) {
          const content = await readFile(f, 'utf8');
          const matches = content.match(FORBIDDEN);
          if (matches) offenders.push({ file: f, matches });
        }
        expect(offenders, JSON.stringify(offenders, null, 2)).toEqual([]);
      });
      ```

- **Task 2.4 — Add `scripts/lint-force-dynamic.sh` gate for authenticated layouts**
  - **Files:** `scripts/lint-force-dynamic.sh`
  - **Action:**
    - Create the script per RESEARCH §G:
      ```bash
      #!/usr/bin/env bash
      set -euo pipefail
      EXPECTED='export const dynamic = .force-dynamic.'

      mapfile -t LAYOUTS < <(find app -path '*/(authenticated)/*' -name 'layout.tsx')

      if [[ ${#LAYOUTS[@]} -eq 0 ]]; then
        echo "ERROR: No (authenticated) layouts found. Expected at least one."
        exit 1
      fi

      for layout in "${LAYOUTS[@]}"; do
        if ! grep -qE "$EXPECTED" "$layout"; then
          echo "ERROR: $layout missing 'export const dynamic = \"force-dynamic\"'"
          exit 1
        fi
      done
      echo "OK: All authenticated layouts have force-dynamic."
      ```
    - `chmod +x scripts/lint-force-dynamic.sh`.
    - Important: the script also fails if **zero** authenticated layouts exist — this catches the regression where Phase 2 accidentally deletes the route group placeholder before adding real auth.
  - **Verification:** `bash scripts/lint-force-dynamic.sh` on the clean tree (after Slice 1 added the placeholder) exits 0 with `OK: All authenticated layouts have force-dynamic.`. Delete the `export const dynamic` line temporarily — script exits 1. Restore.
  - **Commit:** `feat(ci): add scripts/lint-force-dynamic.sh enforcing force-dynamic on every authenticated layout`

- **Task 2.5 — Add `scripts/lint-arabic-wrapper.sh` soft-warn gate for raw Arabic literals**
  - **Files:** `scripts/lint-arabic-wrapper.sh`
  - **Action:**
    - Create the script per RESEARCH §E Option (a):
      ```bash
      #!/usr/bin/env bash
      # Phase 1 soft warning: flags Arabic Unicode block (U+0600–U+06FF) characters
      # in src/ + app/ source. Reviewer manually confirms each is inside <ArabicText>
      # or in an i18n message catalog. AST-level enforcement deferred to Phase 4
      # when content lands (RESEARCH §E).
      set -euo pipefail
      OFFENDERS=$(rg --type tsx --type ts '[؀-ۿ]' src/ app/ --no-heading --line-number 2>/dev/null || true)

      if [[ -n "$OFFENDERS" ]]; then
        echo "WARN: Arabic literals found in source. Verify each is inside <ArabicText> or i18n message catalog:"
        echo "$OFFENDERS"
        # Phase 1: soft fail (exit 0 with warning). Phase 4 will harden to exit 1.
        exit 0
      fi
      echo "OK: No raw Arabic literals in source."
      ```
    - `chmod +x scripts/lint-arabic-wrapper.sh`.
    - The exit-0-with-warning shape is intentional for Phase 1 — RESEARCH §E is explicit that a regex can't distinguish good wraps from bad. We let the script print findings so reviewers can eyeball them during PR review. Phase 4 will harden this to AST-level enforcement.
  - **Verification:** `bash scripts/lint-arabic-wrapper.sh` prints any Arabic literals (e.g., the one in `messages/ar.json` won't match because the script restricts to `.ts`/`.tsx`, not JSON — confirmed in the script via `--type tsx --type ts`). Exit code is 0 in both clean and dirty states.
  - **Commit:** `feat(ci): add scripts/lint-arabic-wrapper.sh soft-warn gate for raw Arabic literals in source`
