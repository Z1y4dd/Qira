  - **Files:** `src/lib/sdk-allowlist.ts`
  - **Action:**
    - Create the file per RESEARCH §G lines 745–768:
      ```ts
      /**
       * Single source of truth for third-party origins authorized on child-facing
       * routes. COMP-LEGAL-01: nothing outside this list may receive a request
       * from any Qira surface. Adding to this list requires:
       *   1. Documenting WHY in this file (purpose, data sent, retention, sub-processors).
       *   2. Confirming the origin is not a behavioral-tracking surface.
       * Enforced at runtime by tests/e2e/network-audit.spec.ts.

        /\.supabase\.co$/,    // production project URLs
        /\.supabase\.in$/,    // staging Supabase region
      ] as const;
      ```
    - Do NOT include localhost or 127.0.0.1 patterns — the Playwright test in Task 5.4 handles same-origin separately (request URL host equals page URL host → skip).
    - Add a comment block listing the explicitly-FORBIDDEN hosts to make the intent visible in code review (analytics, ad networks, session replay, font CDNs):
      ```
      // EXPLICITLY FORBIDDEN (no entries below — listed for reviewer reference):
      //   fonts.googleapis.com, fonts.gstatic.com  (FOUND-03 — fonts are self-hosted)
      //   *.google-analytics.com, *.googletagmanager.com  (no analytics on child routes)
      //   *.segment.com, *.amplitude.com, *.mixpanel.com  (no behavioral tracking)
      //   *.hotjar.com, *.fullstory.com, *.logrocket.com  (no session replay on kids)
      //   *.doubleclick.net, *.adservice.google.com  (no ad networks)
      //   *.posthog.com, *.heap.io                  (no third-party event sinks)
      //   *.facebook.net, connect.facebook.net      (no Meta pixels)
      ```

- **Task 5.2 — Sanity-test the allow-list shape**
  - **Files:** `tests/invariants/sdk-allowlist.test.ts`
  - **Action:**
    - Create a tiny Vitest that asserts the structure:
      ```ts
      import { ALLOWED_HOST_PATTERNS } from '@/lib/sdk-allowlist';
      import { describe, test, expect } from 'vitest';

      describe('SDK allow-list', () => {
        test('contains only Supabase host patterns', () => {
          for (const p of ALLOWED_HOST_PATTERNS) {
            expect(p.source).toMatch(/supabase/);
          }
        });
        test('rejects known-forbidden hosts', () => {
          const FORBIDDEN = ['fonts.googleapis.com', 'www.google-analytics.com', 'cdn.segment.com', 'static.hotjar.com', 'connect.facebook.net'];
          for (const host of FORBIDDEN) {
            expect(ALLOWED_HOST_PATTERNS.some((p) => p.test(host))).toBe(false);
          }
        });
        test('accepts known-allowed Supabase hosts', () => {
          const OK = ['abcdef.supabase.co', 'project-xyz.supabase.in'];
          for (const host of OK) {
            expect(ALLOWED_HOST_PATTERNS.some((p) => p.test(host))).toBe(true);
          }
        });
      });
      ```
  - **Verification:** `pnpm test:run` reports the three new assertions passing.
  - **Commit:** `test(invariants): sanity-check SDK allow-list shape against Supabase and known-forbidden hosts`
