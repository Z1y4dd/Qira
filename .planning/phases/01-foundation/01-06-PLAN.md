# Plan 06: Deploy + Verify

## Goal

Provision the cloud Supabase project, apply the Drizzle migration against it, link the GitHub repo to Vercel, deploy `main`, and prove all five Phase 1 ROADMAP success criteria simultaneously by re-running the Playwright suite against the live URL and inspecting the Network tab manually.

## Demo gate

A public Vercel URL (e.g., `https://qira-<hash>.vercel.app/`) responds:
1. `curl -sI https://qira-<hash>.vercel.app/` → `HTTP/2 307` with `Location: /ar` (the middleware).
2. `curl -s https://qira-<hash>.vercel.app/ar/` returns HTML containing `lang="ar"`, `dir="rtl"`, and the welcome literal.
3. `PLAYWRIGHT_BASE_URL=https://qira-<hash>.vercel.app pnpm e2e` passes all 4 projects.
4. Opening the URL in a real browser → DevTools Network tab, filter to "Font" → all results are served from `qira-<hash>.vercel.app/_next/static/media/*.woff2` (same-origin). Zero rows for `fonts.googleapis.com` or `fonts.gstatic.com`.
5. `psql "$PRODUCTION_DATABASE_URL" -c "\dt public.*"` lists 8 tables; `pnpm db:seed` (against production DATABASE_URL) inserts 20 `levels` rows; re-run is idempotent.

## Tasks

- **Task 6.1 — Create the cloud Supabase project and capture connection details**
  - **Files:** none (manual cloud setup; values go into Vercel env vars in Task 6.3)
  - **Action:**
    - In the Supabase Dashboard, create a new project named `qira-v1`. Pick the region closest to the primary diaspora user base (recommendation: `eu-central-1` for US+UK+Europe coverage; `us-east-1` if US-heavy).
    - Set a strong database password and save it in your password manager.
    - From Project Settings → Database → Connection string, capture:
      - The **pooler URL** (transaction-mode, port 6543) — this is `DATABASE_URL` for runtime.
      - The **direct URL** (port 5432) — this is `DIRECT_DATABASE_URL` for migrations.
    - From Project Settings → API, capture: project URL, anon key, service-role key.
    - Confirm the project's connection pooling is set to "Transaction" mode (default).
  - **Verification:** `psql "$DIRECT_DATABASE_URL" -c "SELECT 1"` returns `1`. `psql "$DATABASE_URL" -c "SELECT 1"` returns `1` (pooler accepts simple selects).
  - **Commit:** none — the file changes that lock these decisions live in `.env.example` (already there) and the Vercel env vars (Task 6.3). Skip the commit step for this task.

- **Task 6.2 — Apply the Drizzle migration to the cloud project and seed levels**
  - **Files:** none (DB-state-only change; the migration SQL is already committed from Slice 3)
  - **Action:**
    - Export the cloud `DIRECT_DATABASE_URL` into your local shell (do NOT use `DATABASE_URL` — pooler doesn't accept DDL).
    - Run `pnpm db:migrate`. Expected output: applies migration `0000_initial`. If the migration log already shows applied (idempotent runs), confirm the schema with `psql "$DIRECT_DATABASE_URL" -c "\dt public.*"` → 8 tables.
    - Run the RLS sanity SQL: `psql "$DIRECT_DATABASE_URL" -c "SELECT tablename FROM pg_tables WHERE schemaname = 'public' AND tablename NOT IN (SELECT tablename FROM pg_policies WHERE schemaname = 'public');"` → zero rows.
    - Export `DATABASE_URL` (pooler) and run `pnpm db:seed`. Expected: `Seeded 20 level rows.`
  - **Verification:** `psql "$DIRECT_DATABASE_URL" -c "SELECT count(*) FROM levels"` returns `20`. Repeat `pnpm db:seed` and observe `Seeded 0 level rows.` (idempotent — `onConflictDoNothing` working).
  - **Commit:** none.

- **Task 6.3 — Link Vercel project and configure environment variables**
  - **Files:** `vercel.json` (optional — only if framework auto-detection needs an override), `.vercel/` directory (auto-generated by `vercel link`; add to `.gitignore`)
  - **Action:**
    - `pnpm dlx vercel@latest login` (one-time).
    - `pnpm dlx vercel@latest link` — link the repo to a new or existing Vercel project. Choose the `pnpm` build preset (Vercel auto-detects from `pnpm-lock.yaml`).
    - Set production env vars via `pnpm dlx vercel env add <NAME> production`, one per:
      - `DATABASE_URL` (pooler)
      - `DIRECT_DATABASE_URL` (direct — Vercel won't use this at runtime but it's set for migrations from GitHub Actions if added later)
      - `NEXT_PUBLIC_SUPABASE_URL`
      - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
      - `SUPABASE_SERVICE_ROLE_KEY` (server-only — Vercel handles this scoping automatically)
    - Also add the same vars to `preview` scope for branch deploys.
    - Add `.vercel/` to `.gitignore` (Vercel CLI generates a `.vercel/project.json` local file — don't commit).
    - If Next 16 + Vercel auto-detect doesn't pick up `pnpm`, create `vercel.json`: `{ "buildCommand": "pnpm build", "installCommand": "pnpm install --frozen-lockfile" }`. Otherwise skip.
  - **Verification:** `pnpm dlx vercel env ls production` lists the five vars. `cat .gitignore | grep -q '^.vercel'`.
  - **Commit:** `chore(deploy): link Vercel project and configure production env vars` — only if `vercel.json` or `.gitignore` changes are present; otherwise none.

- **Task 6.4 — Deploy `main` and run the full verification suite against the live URL**
  - **Files:** none (deploy is a CLI action; verification artifacts captured in Task 6.5)
  - **Action:**

    - Capture the production URL (e.g., `https://qira-<hash>.vercel.app/`).
    - Run the verification against the live URL:

      - `curl -s https://qira-<hash>.vercel.app/ar/ | grep -E 'lang="ar"|dir="rtl"'` → both attributes present.
      - `PLAYWRIGHT_BASE_URL=https://qira-<hash>.vercel.app pnpm e2e --project=chromium-desktop` → green.
      - Repeat for `webkit-desktop`, `chromium-mobile`, `webkit-mobile`. All green.
    - Open the live URL in a desktop browser. DevTools → Network → filter "Font". Confirm every result's Domain column equals the Vercel host. Filter "All" → confirm zero rows for `fonts.googleapis.com`, `fonts.gstatic.com`, `google-analytics.com`, `segment.com`, `hotjar.com`, `mixpanel.com`, `posthog.com`. The only non-same-origin host should be... none (Phase 1 makes no Supabase calls from the landing page — Supabase hits start in Phase 2).
  - **Verification:** All four Playwright projects pass against the live URL. DevTools Network audit manually confirmed zero forbidden hosts. The `<html lang="ar" dir="rtl">` is in the live HTML response.
  - **Commit:** none (no file changes).
