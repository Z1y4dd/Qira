## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-escape-hatch-bypass | Tampering | abortPlacementAction reason argument | mitigate | Task 5.1 adds a runtime literal-union check (`reason !== 'too_hard' && reason !== 'too_easy'` throws). The service-layer `abortPlacement` (Plan 03) also re-validates internally as defense-in-depth. |
| T-3-rls-cross-parent | Information Disclosure / Elevation | attemptId argument | mitigate | The service's `abortPlacement` issues UPDATE against `attempts` filtered by `attempts.id = attemptId AND child belongs to active parent`. RLS on `attempts` enforces the parent-ownership check at the DB layer (Phase 1). Forging another parent's attemptId returns "0 rows updated" — the action redirect still fires but no data is corrupted. |
| T-3-bundle-leak | Information Disclosure | EscapeHatch render + RSC wire payload | mitigate | (a) EscapeHatch never reads `choices` or `isCorrect` — zero risk in the component itself. (b) **RUNTIME GATE (BLOCKER 4):** Task 5.3 adds a Playwright `page.waitForResponse()` assertion against the `/placement/{attemptId}` RSC response body — fails the suite if `'isCorrect'`, `'is_correct'`, or the correct-choice ID appears. This complements the grep invariant from Plan 01 Task 1.5 (`tests/invariants/placement-bundle-leak.test.ts`): grep catches code-level leaks; the Playwright assertion catches runtime serialization bugs (e.g., Drizzle accidentally returning the full row, or a stale type annotation). Both are required per RESEARCH §"Validation Architecture Pillar 2". |
