### Task 1.1 — Extract deterministicShuffle and fix ArabicText diacritics prop

**Type:** refactor

**Files:**
`src/services/_shuffle.ts`,
`src/services/placement.ts`,
`src/components/arabic-text.tsx`,

**Action:**

1. Create `src/services/_shuffle.ts`. Copy the `deterministicShuffle` function verbatim from `src/services/placement.ts:157-166` into this new file. Export it as a named export. The file should import `createHash` from `node:crypto` — that is the only import it needs. Add the same TSDoc comment that placement.ts carries: "Deterministic shuffle using SHA-256 hash of the seed. Same seed always produces the same order (D-02 retake reproducibility). Uses successive 4-byte windows of the hex digest as sort keys."

2. In `src/services/placement.ts`: replace the inline `deterministicShuffle` function body (lines 157-166) with an import from `./_shuffle` and a re-export for backward compatibility. The re-export line must read: `export { deterministicShuffle } from './_shuffle';` The `import { createHash }` at line 12 of placement.ts can be removed after the extraction (the function no longer lives in placement.ts). All other placement.ts code is unchanged.

3. In `src/lib/arabic.ts` (create this file — it does not yet exist): add a single exported function `stripTashkeel(text: string): string`. The implementation applies NFC normalization first (`text.normalize('NFC')`), then strips Arabic diacritics using the regex `/[ً-ْٰ]/g` (covers Fathah through Sukun plus Dagger Alef). Add a JSDoc comment: "Strips Arabic Tashkeel (diacritics) from a NFC-normalized string. Used server-side to produce a diacritics-free version of passage text. Do NOT use CSS `font-feature-settings` for this — that approach breaks Arabic letter joining (liga off disables contextual alternates, not just diacritics)."

4. In `src/components/arabic-text.tsx`: the current `diacriticsHideClass` at line 25 is `'[font-feature-settings:"liga"_off]'`. This is wrong — disabling `liga` suppresses contextual alternates (letter joining), not Tashkeel visibility. Per pre-resolved decision #4, the correct approach for the toggle is server-side `stripTashkeel()`, not a CSS class. Change the approach: remove `diacriticsHideClass` constant entirely and remove the `diacritics === 'hide' && diacriticsHideClass` conditional from the `cn()` call. The `diacritics` prop will remain in the interface for forward-compatibility but the hide path no longer applies a class — it is a no-op on the component. Add an inline JSDoc comment above the prop: "Tashkeel toggling is handled server-side via `stripTashkeel()` in `src/lib/arabic.ts`, not via a CSS class. This prop is preserved for typing convenience; the server pre-processes the text before passing it to this component." The component otherwise remains identical: same `sizeClasses`, same `<bdi>` wrapper, same `lang="ar"`.
