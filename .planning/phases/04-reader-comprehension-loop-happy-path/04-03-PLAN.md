Action 3: `recordComprehensionAnswerAction({ attemptId: string, questionId: string, chosenChoiceId: string })` — stub placeholder only in Plan 03; fully wired in Plan 04. Write the function signature and the auth guard, but stub the service call body with `throw new Error('recordComprehensionAnswerAction: wired in Plan 04-04')`. This lets the route tree compile.

**`page.tsx`** (auto-route, the entry to the reader) — runs server-side:
- `export const dynamic = 'force-dynamic'`.
- Auth guard: `cookies() → createClient → requireActiveChild → catch AuthError → redirect('/choose-child')`.
- Call `getNextTextForChild(active.id)`.
- If null: render a simple Arabic message "لا توجد نصوص متاحة في مستواك حالياً" wrapped in `<ArabicText>` inside the layout. This is the Phase 4 empty-state (no retry polish — that is Phase 4.1 scope).

To keep the page stateless (no double-start), use the Server Action inline — call `startComprehensionAttempt({ childId: active.id, textId: text.id })` directly in the page (this is a server component, not a Client Component; the service call is safe here) and `redirect(\`/reader/${attemptId}\` as Route)`.

Note on why actions.ts also has `startReadingAttemptAction`: the action exists for potential use from a CTA button (if needed in Phase 4.1). The page.tsx auto-redirects immediately without a button — this mirrors how `placement/start/page.tsx` renders a "Start" form, but the reader skips the CTA for the happy path. Both paths are safe.

### Task 3.2 — Passage screen: passage-renderer + Tashkeel toggle + [attemptId] page (passage branch)

**Type:** feature

**Files:**
`src/components/reader/passage-renderer.tsx`,
`src/components/reader/tashkeel-toggle.tsx`,
`src/app/(authenticated)/(active)/(placement-gate)/reader/[attemptId]/page.tsx`

**Action:**

**`src/components/reader/passage-renderer.tsx`** — Server Component (no `'use client'`):

Props:
```typescript
interface PassageRendererProps {
  titleAr: string;
  bodyAr: string;
  /** Tashkeel default: 'on' for levels 1-10, 'off' for 11-20. */
  tashkeelDefault: 'on' | 'off';
  /** Runtime override from ?tashkeel URL param. Undefined = use tashkeelDefault. */
  tashkeelOverride?: 'on' | 'off';
  attemptId: string;
}
```

Rendering:
- Compute `showTashkeel = tashkeelOverride ?? tashkeelDefault`.
- The `bodyText` passed to the Arabic text component is: if `showTashkeel === 'off'`, call `stripTashkeel(bodyAr)` (import from `@/lib/arabic`). If `showTashkeel === 'on'`, use `bodyAr` as-is.
- Render title as `<ArabicText as="h1" size="reader" className="text-3xl mb-6 block text-start">` (import from `@/components/arabic-text`).
- Render body as `<ArabicText as="p" size="reader" data-testid="passage-body" className="text-start leading-[1.9] mb-8 block">`. The `data-testid="passage-body"` is mandatory (LIB-04 Playwright visual regression target).
- Below the body, render the `<TashkeelToggle>` component passing `current={showTashkeel}` and `attemptId`.
- Below the toggle, render the "أنا جاهز" button: a `<form action={advanceToQuestionsAction.bind(null, { attemptId })}>` with a submit button styled `w-full min-h-14 rounded-lg bg-primary text-primary-foreground text-xl`. Import `advanceToQuestionsAction` from `@/app/(authenticated)/(active)/(placement-gate)/reader/actions`.
- The outer div uses `dir="rtl"` and `className="space-y-4"`.

Note: `<ArabicText size="reader">` already sets `font-naskh text-2xl leading-[1.9]` (arabic-text.tsx:20). Line-height is 1.9 which is ≥ 1.8 (LIB-03 requirement). Do not override it.

**`src/components/reader/tashkeel-toggle.tsx`** — `'use client'` component:

This is the interactive Tashkeel toggle. Per pre-resolved decision #5, toggling re-renders via `?tashkeel=on|off` URL param (server re-render), not client state. The component uses `useRouter` and `useSearchParams` from `next/navigation`.

Props:
```typescript
interface TashkeelToggleProps {
  current: 'on' | 'off';
  attemptId: string;
}
```

Render:
- A small toggle button labeled in Arabic: when `current === 'on'` show "إخفاء التشكيل", when `'off'` show "إظهار التشكيل". Use `<ArabicText size="caption">` for the label.
- `onClick`: call `router.push(\`/reader/${attemptId}?tashkeel=${current === 'on' ? 'off' : 'on'}\`)`. This causes the page to re-render server-side with the new tashkeel state.
- Style: `button` with `variant="outline"` visuals — `className="text-sm px-3 py-1 rounded border border-muted-foreground text-muted-foreground hover:bg-muted"`. Use logical `px-` not `pl-/pr-`.
- Wrap in a `<div className="flex justify-start mb-4">` (RTL-aware: `justify-start` = right-aligned in RTL layout).

**`src/app/(authenticated)/(active)/(placement-gate)/reader/[attemptId]/page.tsx`** — Server Component:

This page handles BOTH the passage screen (default) and the question screen (showPassage=0). In Plan 03 we implement only the passage branch; the question branch is a stub that Plan 04 fills.

```typescript
export const dynamic = 'force-dynamic';

interface ReaderAttemptPageProps {
  params: Promise<{ attemptId: string }>;
  searchParams: Promise<{ showPassage?: string; tashkeel?: string }>;
}
```

Implementation:
- `const { attemptId } = await params`.
- `const { showPassage, tashkeel } = await searchParams`.
- Auth guard: `cookies() → createClient → requireActiveChild → catch AuthError → redirect('/choose-child')`.
- Load the attempt: SELECT `id, childId, textId, finishedAt` FROM `attempts` WHERE `id = attemptId AND kind = 'reading'`. If not found or `childId !== active.id`, `redirect('/reader' as Route)` (defense-in-depth cross-child isolation, mirrors result page pattern in `placement/[attemptId]/result/page.tsx:58-61`).
- If `finishedAt` is set, `redirect(\`/reader/${attemptId}/result\` as Route)`.
- Call `getTextWithQuestions(attempt.textId as TextId, attemptId as AttemptId)`.
- Resolve `tashkeelOverride`: if `tashkeel === 'on'`, set `'on'`; if `tashkeel === 'off'`, set `'off'`; else `undefined`.

**Passage branch** (`showPassage` is undefined or not `'0'`):
- Render `<PassageRenderer titleAr={text.titleAr} bodyAr={text.bodyAr} tashkeelDefault={text.tashkeelDefault} tashkeelOverride={tashkeelOverride} attemptId={attemptId} />`.

**Question branch** (`showPassage === '0'`):
- Render a placeholder: `<div data-testid="question-screen-stub"><ArabicText>الأسئلة — تأتي في الخطة التالية</ArabicText></div>`. Plan 04 replaces this with the real question flow.

Import `getTextWithQuestions` from `@/services/comprehension`. Import `PassageRenderer` from `@/components/reader/passage-renderer`. Import needed DB tables from `@/db` via a direct Drizzle query (same pattern as the result page in placement).

**AC:**
- Visiting `/reader/[attemptId]` (with a valid reading attempt) renders the passage with `data-testid="passage-body"` present in the DOM.
- The Tashkeel toggle button is visible.
- Clicking "إخفاء التشكيل" navigates to `?tashkeel=off` and the passage body no longer contains Arabic diacritics.
- The "أنا جاهز" button navigates to `?showPassage=0`.
- `pnpm tsc --noEmit` passes.
- `grep "font-feature-settings" src/components/reader/` returns empty (the wrong diacritics mechanism is absent from reader components).
- `pnpm biome check src/components/reader/ src/app/\(authenticated\)/\(active\)/\(placement-gate\)/reader/` passes.

### Task 3.3 — Draft reader-visual.spec.ts snapshot baseline

**Type:** test

**Files:**
`tests/e2e/reader-visual.spec.ts`

**Action:**

Replace the stub from Plan 04-01 with a partial visual regression spec. Plan 04-05 will add more snapshot scenarios; this plan establishes the baseline for the passage screen.

```typescript
import { expect, test } from '@playwright/test';
import { createTestParent, deleteTestParent, type TestParent } from './_helpers/test-parents';
```

The spec is `test.describe.serial('reader visual regression', () => { ... })` with `beforeAll` / `afterAll` that create and delete a test parent.

Test: `'passage screen — Tashkeel ON (Level 5) renders Noto Naskh at ≥ 1.9 line-height'`:
- Sign in as the test parent, create a child, select child (triggers placement gate → redirect to placement/start).
- Complete the placement assessment (answer all 15 questions by clicking the first choice each time — copy the loop-and-click pattern from `placement-flow.spec.ts:103-131`). Wait for `toHaveURL(/\/placement\/.*\/result/)`.
- Navigate to `/reader`. Wait for redirect to `/reader/[uuid]`.
- Wait for `[data-testid="passage-body"]` to be visible.
- Verify line-height via `page.evaluate`: `document.querySelector('[data-testid="passage-body"]')?.computedStyleMap().get('line-height')`. Assert the numeric value is >= 1.8 times the font-size. Alternatively: `const style = await page.locator('[data-testid="passage-body"]').evaluate(el => window.getComputedStyle(el).lineHeight)` — assert that the line-height value is ≥ 44px at 24px base (1.9 × 24 = 45.6 — accept >= 44 to tolerate subpixel rounding).
- Take a visual snapshot: `await expect(page).toHaveScreenshot('reader-passage-tashkeel-on.png', { fullPage: false })`. This anchors the Noto Naskh + diacritics rendering.

Test: `'Tashkeel toggle navigates to ?tashkeel=off and strips diacritics from passage body'`:
- Navigate to the passage page (reuse state from previous test via shared `let readerUrl: string`).
- Click the toggle button (text "إخفاء التشكيل").
- Assert URL contains `tashkeel=off`.
- Assert `[data-testid="passage-body"]` text does NOT contain Arabic diacritics. Use `page.locator('[data-testid="passage-body"]').innerText()` and check that no characters in the Unicode range `ً-ْ` (Fathah through Sukun) and `ٰ` (Dagger Alef) are present.

Mark both tests with a comment: `// Snapshot baselines updated with: pnpm e2e:update-snapshots`.

**AC:**

- After `pnpm e2e:update-snapshots`, a subsequent `pnpm e2e --grep "reader visual regression"` passes.
